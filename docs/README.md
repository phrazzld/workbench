# Documentation

This directory contains documentation for project tracking, terminal setup, and professional resources.

## Structure

- **[projects.md](projects.md)** - Project registry
- **[project-names.md](project-names.md)** - Available and assigned project names
- **[PROJECTS_GUIDE.md](PROJECTS_GUIDE.md)** - Workflow for maintaining the project registry
- **[WEZTERM.md](WEZTERM.md)** - WezTerm configuration notes
- **[professional/](professional/)** - Professional resources like resume and CV templates

## Purpose

The documentation in this directory serves several key functions:

1. Tracking project status and naming decisions
2. Documenting terminal and local development setup
3. Keeping professional materials close to the rest of the personal workbench

## Usage

These documents should be consulted when:

- Updating the project registry
- Picking or retiring project names
- Rebuilding terminal configuration
- Updating resume or CV materials

## Maintenance

All documentation should be kept up-to-date as the workbench evolves.
